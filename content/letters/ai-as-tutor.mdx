# 把 AI 当家教用：写给不会用 ChatGPT 的同学

入学没多久就有人问我："学长，AI 到底怎么用？我点开 ChatGPT 不知道问啥。"

这封信不讲那种 *"10 个让你效率翻倍的 prompt"* 的东西。**你不需要 prompt 模板**，你需要的是把 AI 放在大学生活里**用对位置**。

## 它是干什么的，先想清楚

把 AI 想成一个**24 小时在线、有点啰嗦、什么都懂一点但不一定都对**的家教。它的优点和短板都很明确：

| 它擅长 | 它不擅长 |
| --- | --- |
| 把一段你看不懂的内容**换种说法解释** | 替你输出"对的答案"——它会很自信地编 |
| 帮你**把模糊的想法整理成结构** | 给你**最新的、特定学校的**信息（比如教务系统怎么操作） |
| **挑你写的东西的毛病**（语法、逻辑、代码） | 替你做你本来该练的基本功 |
| 解释一个概念、举几个例子 | 数学严格证明、复杂代数推导（容易错） |

> 一句话：**让它当陪练，不要让它当代笔**。

## 三个能立刻提升体验的用法

### 1. 预习时——让它把课本"翻译"一遍

最痛苦的预习场景是：打开教材，第一段就看不懂术语。这时候不要硬啃，把那段话整段贴给 AI，加一句：

```
我是大一学生，没学过 X，你能用我能听懂的话把这段重新讲一遍吗？
最后给我留三个能检验我有没有真懂的小问题。
```

它给的解释**不一定权威**，但**够你建立第一印象**。等老师讲的时候你就有个钩子可以挂。

### 2. 写报告 / 论文时——让它当审稿人

**严禁让它替你写**。两个原因：

1. 学校查重和 AI 检测越来越狠，赌不起
2. 你交上去的东西是你的脸，糊弄一次老师就记住你了

**正确用法**是写完之后扔给它：

```
这是我写的实验报告，请假装你是一个严格的助教，
指出三个最严重的问题，但不要替我重写。
```

让它**指出问题**比让它**给答案**对你成长更有用。

### 3. 编程作业卡住时——让它解释报错

> ImportError: cannot import name 'xxx' from partially initialized module 'yyy'

新生看到这玩意一头雾水，正常。直接复制报错 + 你的代码片段贴给 AI，问：

> 这个错为什么会发生？我在写一个 Python 模块，刚开始学。

它会给你**最常见的几个原因**。然后你自己去查、自己改——**改完跑通了**才是你学会了。

## 三条红线

1. **考试期间用 AI 替考——退学起步**。教育部 2024 年起把这类行为定性为学术不端的最高级别。
2. **毕业论文整段 AI 生成——延毕风险极大**。现在大部分高校都要求过 AIGC 检测。
3. **把私人信息（身份证、银行卡、私密对话）发给 AI**——会被记录、可能用于训练。**永远不要**。

## 选哪个 AI 不重要

ChatGPT、Claude、Gemini、DeepSeek、豆包——**对学生来说，区别没有你想的大**。挑一个能稳定访问、UI 顺手的就行。具体到工具列表，看 [AI 工具](/ai) 那一页。

国内能直接用的、不要钱的有几个，都先注册，遇到一个崩了切下一个。
